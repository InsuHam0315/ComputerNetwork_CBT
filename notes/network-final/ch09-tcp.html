1. 챕터 개요

TCP는 전송 계층에서 가장 중요한 연결형 신뢰성 프로토콜이다. 시험에서는 TCP의 특징, 헤더 필드, 3-way handshake, 연결 해제, 흐름 제어, 오류 제어, 혼잡 제어가 거의 한 묶음으로 출제될 수 있다.

TCP는 응용 프로그램에는 바이트 스트림처럼 보이지만, 실제 전송은 세그먼트 단위로 이루어진다. 순서 번호와 확인 응답 번호는 이 바이트 스트림을 안정적으로 관리하기 위한 핵심 필드이다.

흐름 제어와 혼잡 제어는 반드시 구분해야 한다. 흐름 제어는 수신자의 버퍼와 처리 능력, 혼잡 제어는 네트워크 내부 상태를 기준으로 송신량을 조절한다.

2. 시험에 나오는 핵심

  • TCP는 연결형, 신뢰성, 순서 보장, 양방향 전송을 제공하는 전송 계층 프로토콜이다.
  • TCP 헤더의 포트 번호는 응용 프로세스를 구분한다.
  • 순서 번호와 확인 응답 번호는 데이터 순서와 수신 확인을 관리한다.
  • 3-way handshake는 SYN → SYN+ACK → ACK 흐름으로 연결을 설정한다.
  • 연결 해제에는 FIN과 ACK가 사용되며 양방향 종료 과정을 구분할 수 있다.
  • TCP 제어 기능은 흐름 제어, 오류 제어, 혼잡 제어로 나누어 정리한다.

3. 핵심 개념 상세 설명

TCPTransmission Control Protocol이다. IP 위에서 동작하며 신뢰성 있는 전송 서비스를 제공한다.
세그먼트TCP가 데이터를 전송할 때 사용하는 단위이다. IP 계층으로 내려가면 세그먼트가 IP 패킷에 실린다.
포트 번호호스트 안의 응용 프로세스를 구분한다. 출발지 포트와 목적지 포트가 TCP 헤더에 포함된다.
소켓IP 주소와 포트 번호를 결합해 통신 종단점을 식별하는 개념으로 설명될 수 있다.
순서 번호전송되는 바이트 스트림의 위치를 나타낸다. 수신자는 순서 번호로 데이터 순서와 누락을 판단할 수 있다.
확인 응답 번호다음에 받기를 기대하는 바이트 번호를 나타내는 방식으로 설명된다. 누적 ACK 개념과 연결된다.
SYNTCP 연결 설정을 시작하고 초기 순서 번호를 동기화하는 플래그이다.
ACK수신 확인을 나타내는 플래그이자 확인 응답 번호와 함께 사용된다.
FIN더 이상 보낼 데이터가 없음을 알리고 연결 종료를 요청하는 플래그이다.
RST비정상 상황에서 연결을 즉시 재설정할 때 사용되는 플래그이다.
3-way handshake클라이언트 SYN, 서버 SYN+ACK, 클라이언트 ACK 순서로 연결을 설정한다.
연결 해제TCP는 양방향 데이터 흐름을 각각 닫아야 하므로 FIN/ACK 교환이 여러 단계로 이루어질 수 있다.
흐름 제어수신 윈도우를 사용해 수신자가 감당할 수 있는 만큼만 보내도록 조절한다.
오류 제어체크섬, ACK, 타이머, 재전송, 중복 ACK 등을 이용해 손실과 오류를 복구한다.
혼잡 제어네트워크 혼잡을 고려해 혼잡 윈도우를 조절한다. 느린 시작, 혼잡 회피, 빠른 재전송 같은 용어와 연결된다.
ECN혼잡을 반드시 패킷 손실로만 알리지 않고, 네트워크 장비가 혼잡 징후를 표시해 송신자가 속도를 줄이도록 돕는 방식으로 정리한다.

5. 계층별 기능 정리

IP와 TCPIP는 호스트 간 비연결형 전달을 제공하고, TCP는 그 위에서 연결형 신뢰성 전송을 제공한다.
응용 계층과 TCP웹, 메일, 파일 전송 같은 응용은 TCP 연결을 이용해 안정적인 바이트 스트림을 사용할 수 있다.
포트 기반 전달IP 주소가 호스트를 찾는다면 포트 번호는 호스트 안의 응용 프로세스를 찾는다.

8. 모두 고르기 예상 선지

옳은 선지로 만들기 쉬운 문장

  • TCP는 연결형 전송 서비스를 제공한다.
  • TCP는 순서 번호와 ACK를 이용해 신뢰성 있는 전송을 지원한다.
  • 3-way handshake는 연결 설정 절차이다.
  • 흐름 제어는 수신 윈도우와 관련된다.
  • 혼잡 제어는 네트워크 과부하 완화와 관련된다.

함정 선지로 나오기 쉬운 문장

  • TCP를 비연결형이고 신뢰성 제어가 없는 프로토콜이라고 하는 선지
  • 흐름 제어와 혼잡 제어를 완전히 같은 기능이라고 하는 선지
  • SYN을 연결 종료 전용 플래그라고 하는 선지
  • TCP 세그먼트를 데이터 링크 계층의 프레임과 같은 계층 단위라고 하는 선지

9. 설명형 예상문제와 모범답안

TCP 3-way handshake를 설명하시오.

클라이언트가 SYN을 보내 연결 설정을 요청하고, 서버가 SYN+ACK로 응답하며, 클라이언트가 ACK를 보내 연결을 확정한다. 이 과정에서 초기 순서 번호를 동기화한다.

핵심 키워드: SYN, SYN+ACK, ACK, 초기 순서 번호

TCP의 흐름 제어와 혼잡 제어를 비교하시오.

흐름 제어는 수신자의 버퍼와 처리 능력을 기준으로 송신량을 조절하고, 혼잡 제어는 네트워크 내부 혼잡 상태를 기준으로 송신 속도를 조절한다.

핵심 키워드: 흐름 제어, 혼잡 제어, 수신자, 네트워크

ECN 패킷의 원리를 설명하시오.

ECN은 네트워크에서 혼잡이 발생하기 전에 패킷에 혼잡 징후를 표시해 수신자와 송신자가 이를 알 수 있게 하고, 송신자가 전송률을 줄이도록 돕는 방식이다.

핵심 키워드: ECN, 혼잡, 표시, 전송률

10. 최종 암기 요약

  • TCP는 연결형 신뢰성 바이트 스트림이다.
  • 3-way는 SYN, SYN+ACK, ACK 순서다.
  • 흐름 제어는 수신 윈도우, 혼잡 제어는 혼잡 윈도우다.
  • 순서 번호와 ACK 번호는 TCP 신뢰성의 핵심이다.
