1. 챕터 개요

상위 계층은 세션 계층, 표현 계층, 응용 계층을 중심으로 정리한다. 실제 인터넷 구현에서는 기능이 합쳐져 보일 수 있지만, 시험에서는 OSI 계층별 기능을 구분해 쓰는 것이 중요하다.

세션 계층은 대화 제어, 토큰, 동기점이 핵심이다. 표현 계층은 데이터 표현 형식, 압축, 암호화가 핵심이다. 응용 계층은 사용자가 직접 접하는 네트워크 서비스를 제공한다.

단답형에서는 표현 계층, 세션 계층, 동기점, 토큰, 추상 문법, 클라이언트-서버 같은 용어가 나오기 쉽다. 설명형에서는 동기점, 표현 계층 기능, 클라이언트-서버 모델을 준비한다.

2. 시험에 나오는 핵심

  • 세션 계층은 통신 주체 사이의 대화 제어와 동기 관리를 담당한다.
  • 토큰은 특정 동작을 수행할 권한을 제어하는 논리 정보로 설명된다.
  • 동기점은 오류 발생 시 복구 기준점으로 사용될 수 있다.
  • 표현 계층은 데이터 표현 형식 변환, 압축, 암호화와 관련된다.
  • 응용 계층은 사용자가 직접 사용하는 네트워크 서비스를 제공한다.
  • 클라이언트-서버 모델은 요청 주체와 서비스 제공 주체를 구분한다.

3. 핵심 개념 상세 설명

세션 계층응용 프로세스 사이의 대화 관계를 설정, 관리, 종료한다. 대화 제어, 토큰 관리, 동기점 관리가 핵심이다.
세션 연결통신 주체 사이에 대화를 진행하기 위해 설정되는 논리적 연결이다. 설정, 유지, 해제 절차와 연결된다.
대화 제어누가 언제 데이터를 보낼 수 있는지, 양방향인지 반이중인지 같은 대화 흐름을 조절한다.
토큰특정 기능을 수행할 수 있는 권한을 나타내는 논리 정보이다. 동시에 수행하면 안 되는 동작을 제어하는 데 쓰일 수 있다.
동기긴 데이터 교환 과정에서 복구 지점을 표시하고, 오류 발생 후 통신 상태를 다시 맞추는 기능이다.
동기점오류가 발생했을 때 처음부터 다시 시작하지 않고 특정 지점부터 재개할 수 있도록 하는 기준점이다. 큰 동기점과 작은 동기점으로 설명될 수 있다.
표현 계층서로 다른 시스템 사이의 데이터 표현 형식 차이를 맞춘다. 코드 변환, 데이터 형식 변환, 압축, 암호화가 대표 기능이다.
추상 문법데이터의 의미 구조를 시스템 독립적으로 표현하는 규칙이다. ASN.1 같은 개념과 연결될 수 있다.
전송 문법실제 전송을 위해 데이터를 어떤 비트열 형태로 인코딩할지 정하는 표현 방식이다.
데이터 표현정수, 문자, 구조체, 이미지 같은 데이터를 서로 다른 시스템이 같은 의미로 해석하도록 맞추는 기능이다.
데이터 압축전송량이나 저장 공간을 줄이기 위해 데이터를 더 작은 형태로 표현한다.
암호화데이터 내용을 보호하기 위해 원래 정보를 알아보기 어려운 형태로 바꾸는 기능이다. 표현 계층 기능으로 설명될 수 있다.
응용 계층사용자가 접하는 네트워크 서비스를 제공한다. 웹, 메일, 파일 전송, 원격 접속 같은 서비스와 연결될 수 있다.
클라이언트-서버 모델클라이언트가 서비스를 요청하고 서버가 요청을 처리해 응답을 제공하는 구조이다.
클라이언트서비스를 요청하는 주체이다. 웹 브라우저처럼 서버에 요청을 보내는 프로그램이 예시가 될 수 있다.
서버요청을 받아 자원이나 처리 결과를 제공하는 주체이다. 웹 서버, 메일 서버, 파일 서버 등이 예시가 될 수 있다.

8. 모두 고르기 예상 선지

옳은 선지로 만들기 쉬운 문장

  • 세션 계층은 대화 제어와 동기점 관리와 관련된다.
  • 토큰은 특정 동작 수행 권한 제어와 연결된다.
  • 표현 계층은 데이터 표현 형식 변환과 압축을 담당할 수 있다.
  • 응용 계층은 사용자에게 네트워크 서비스를 제공한다.
  • 클라이언트는 요청하고 서버는 서비스를 제공한다.

함정 선지로 나오기 쉬운 문장

  • 표현 계층을 라우팅 테이블 계산 전용 계층이라고 하는 선지
  • 동기점이 MAC 주소 변환 기능이라고 하는 선지
  • 응용 계층이 물리 신호 전압만 정의한다고 하는 선지
  • 클라이언트가 항상 서비스를 제공하고 서버가 항상 요청만 한다고 하는 선지

9. 설명형 예상문제와 모범답안

세션 계층에서 동기점의 목적을 설명하시오.

긴 데이터 교환 중 복구 기준점을 만들어 오류가 발생했을 때 처음부터 다시 시작하지 않고 특정 지점부터 재개할 수 있도록 돕는다.

핵심 키워드: 동기점, 복구, 기준점, 재개

표현 계층의 주요 기능을 설명하시오.

서로 다른 시스템 사이의 데이터 표현 형식을 맞추고, 필요하면 압축, 암호화, 코드 변환을 수행한다.

핵심 키워드: 표현 계층, 데이터 표현, 압축, 암호화

클라이언트-서버 모델을 설명하시오.

클라이언트는 서비스를 요청하고 서버는 요청을 처리해 자원이나 결과를 제공하는 통신 모델이다.

핵심 키워드: 클라이언트, 서버, 요청, 제공

추상 문법과 전송 문법의 차이를 설명하시오.

추상 문법은 데이터가 어떤 의미 구조를 갖는지 표현하고, 전송 문법은 그 데이터를 실제 전송 가능한 비트열로 어떻게 인코딩할지 표현한다.

핵심 키워드: 추상 문법, 전송 문법, 의미 구조, 인코딩

10. 최종 암기 요약

  • 세션은 대화, 토큰, 동기점이다.
  • 표현은 형식 변환, 압축, 암호화다.
  • 응용은 사용자 서비스다.
  • 클라이언트는 요청, 서버는 제공이다.
