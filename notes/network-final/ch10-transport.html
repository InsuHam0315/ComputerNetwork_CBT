1. 챕터 개요

전송 계층은 네트워크 계층의 호스트 간 전달을 응용 프로세스 간 전달로 확장한다. 이 장에서는 TCP와 대비되는 UDP, 실시간 전송을 위한 RTP, 그리고 OSI 전송 서비스 관련 개념을 함께 정리한다.

UDP는 비연결형이고 헤더가 단순하다. 이 말은 UDP가 쓸모없다는 뜻이 아니라, 연결 설정과 복잡한 재전송 제어를 줄이고 빠른 전달이 필요한 상황에 적합하다는 뜻이다.

RTP는 실시간 미디어 전송에서 순서 번호와 타임스탬프를 제공한다. RTP가 TCP처럼 완전한 신뢰성 보장을 해 주는 프로토콜이라고 외우면 안 된다.

2. 시험에 나오는 핵심

  • 전송 계층은 포트 번호를 이용해 프로세스 간 통신을 제공한다.
  • UDP는 비연결형 데이터그램 전송 프로토콜이다.
  • UDP 헤더는 출발지 포트, 목적지 포트, 길이, 체크섬 중심으로 단순하다.
  • RTP는 실시간 음성·영상 전송에서 순서와 시간 정보를 제공한다.
  • RTP 릴레이는 실시간 미디어 흐름을 중계하거나 변환하는 역할과 연결된다.
  • OSI TP와 세션 프리미티브는 서비스 기본 동작과 전송 서비스 품질 관점으로 정리한다.

3. 핵심 개념 상세 설명

전송 계층호스트 간 통신을 응용 프로세스 간 통신으로 확장한다. 포트 번호, 다중화, 역다중화가 핵심이다.
포트 번호하나의 호스트 안에서 어떤 응용 프로세스와 통신할지 구분하는 번호이다.
다중화여러 응용 프로세스의 데이터를 전송 계층에서 모아 네트워크 계층으로 내려보내는 기능이다.
역다중화수신 측에서 포트 번호를 보고 데이터를 알맞은 응용 프로세스로 나누어 전달하는 기능이다.
UDPUser Datagram Protocol이다. 연결 설정 없이 데이터그램을 보내며, 헤더가 단순하고 오버헤드가 작다.
UDP 헤더출발지 포트, 목적지 포트, 길이, 체크섬으로 구성된다. TCP의 순서 번호, ACK 번호, 윈도우 같은 필드는 없다.
UDP 체크섬전송 중 오류를 검출하기 위한 값이다. 단순한 프로토콜이라도 오류 검출 정보를 가질 수 있다.
데이터그램UDP에서 독립적으로 취급되는 전송 단위이다. 각 데이터그램은 연결 상태 없이 전달된다.
RTPReal-time Transport Protocol이다. 실시간 오디오·비디오 데이터의 순서, 시간, 페이로드 유형 정보를 제공한다.
RTP 순서 번호패킷 순서 확인과 손실 감지에 도움을 준다. 재생 품질 제어와 연결해 이해한다.
RTP 타임스탬프미디어 데이터의 재생 시점과 동기화를 돕는다. 단순 도착 시간과는 구분된다.
SSRCRTP 스트림의 송신원을 식별하는 값이다. 여러 소스가 섞일 때 구분에 사용된다.
RTP 릴레이미디어 데이터를 중계하거나 변환해 전달 경로를 보조한다. 믹서나 트랜슬레이터 개념과 연결될 수 있다.
세션 프리미티브연결 설정, 데이터 전송, 연결 해제 같은 서비스 기본 동작을 추상적으로 나타내는 명령 또는 연산이다.
OSI TPOSI 전송 프로토콜 계열로, 전송 연결, 오류 회복, 서비스 품질을 계층적으로 다루는 개념이다.

5. 계층별 기능 정리

네트워크 계층과의 관계IP 주소는 호스트를 찾고, 전송 계층 포트 번호는 응용 프로세스를 찾는다.
응용 계층과의 관계응용 프로그램은 TCP 또는 UDP 같은 전송 계층 서비스를 사용해 데이터를 주고받는다.
실시간 전송 관점RTP는 보통 UDP 위에서 동작하며, 실시간 재생에 필요한 순서와 시간 정보를 보조한다.

8. 모두 고르기 예상 선지

옳은 선지로 만들기 쉬운 문장

  • UDP는 비연결형 서비스를 제공한다.
  • UDP 헤더는 TCP보다 일반적으로 단순하다.
  • UDP도 포트 번호를 사용해 응용 프로세스를 구분한다.
  • RTP는 실시간 데이터의 순서와 시간 정보를 제공할 수 있다.
  • 전송 계층은 다중화와 역다중화를 수행한다.

함정 선지로 나오기 쉬운 문장

  • UDP가 3-way handshake를 반드시 수행한다고 하는 선지
  • UDP가 TCP보다 항상 더 강한 신뢰성을 보장한다고 하는 선지
  • RTP를 IP 주소와 MAC 주소 변환 프로토콜로 설명하는 선지
  • 포트 번호를 호스트 자체의 네트워크 계층 주소라고 하는 선지

9. 설명형 예상문제와 모범답안

UDP 프로토콜의 특징을 설명하시오.

UDP는 연결 설정 없이 데이터그램을 보내는 비연결형 전송 프로토콜이다. 헤더가 단순하고 오버헤드가 작아 빠른 전달이 중요한 응용에 적합하지만, TCP와 같은 연결형 신뢰성 제어는 제공하지 않는다.

핵심 키워드: UDP, 비연결형, 단순, 빠른 전송

RTP 헤더의 순서 번호와 타임스탬프 역할을 설명하시오.

순서 번호는 패킷 순서 확인과 손실 판단을 돕고, 타임스탬프는 실시간 미디어의 재생 시점과 동기화를 돕는다.

핵심 키워드: 순서 번호, 타임스탬프, 손실, 재생

전송 계층의 다중화와 역다중화를 설명하시오.

송신 측은 여러 응용 데이터를 포트 번호로 구분해 네트워크로 보내고, 수신 측은 목적지 포트를 보고 데이터를 알맞은 응용 프로세스에 전달한다.

핵심 키워드: 다중화, 역다중화, 포트, 프로세스

10. 최종 암기 요약

  • UDP는 비연결형, 단순 헤더, 낮은 오버헤드다.
  • UDP 헤더는 포트, 길이, 체크섬이다.
  • RTP는 실시간 전송의 순서와 시간을 보조한다.
  • 포트는 프로세스, IP는 호스트를 구분한다.
