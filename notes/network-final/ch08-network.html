1. 챕터 개요

이 장은 IP를 보조하거나 확장하는 네트워크 계층 관련 기술이 많이 나온다. IPv6는 주소와 헤더 구조, ARP/RARP는 주소 변환 방향, ICMP/IGMP는 제어 메시지와 그룹 관리로 구분한다.

IPv6는 128비트 주소와 단순화된 기본 헤더, 확장 헤더 구조가 핵심이다. IPv4와 비교하는 문제에서 주소 길이, 헤더 구조, 체크섬 처리, 확장성 차이가 자주 묶인다.

ARP, RARP, ICMP, IGMP는 약어만 외우면 헷갈리기 쉽다. ARP는 IP에서 MAC, RARP는 MAC에서 IP, ICMP는 오류·진단, IGMP는 멀티캐스트 그룹 관리로 짧게 정리한다.

2. 시험에 나오는 핵심

  • IPv6 주소는 128비트이며 주소 공간이 IPv4보다 훨씬 크다.
  • IPv6 기본 헤더는 단순화하고, 부가 기능은 확장 헤더로 연결할 수 있다.
  • 터널링은 한 프로토콜 패킷을 다른 프로토콜 패킷 안에 캡슐화해 전달한다.
  • ARP는 IP 주소로 MAC 주소를 찾고, RARP는 MAC 주소로 IP 주소를 찾는다.
  • ICMP는 오류 보고와 진단 메시지, IGMP는 멀티캐스트 그룹 관리에 사용된다.
  • 이동 IP는 이동 노드가 네트워크 위치를 바꾸어도 통신을 유지하는 개념과 관련된다.

3. 핵심 개념 상세 설명

IPv6128비트 주소를 사용하는 IP 버전이다. 큰 주소 공간, 단순화된 기본 헤더, 확장 헤더, 자동 설정, 이동성 지원과 연결해 정리한다.
IPv6 주소16비트 단위의 16진수 블록으로 표현한다. 0 압축 표기와 연속된 0 블록 생략 규칙이 출제될 수 있다.
IPv6 기본 헤더Version, Traffic Class, Flow Label, Payload Length, Next Header, Hop Limit, Source Address, Destination Address 등을 포함한다.
확장 헤더IPv6에서 선택 기능을 기본 헤더 밖으로 분리해 필요한 경우에만 연결하는 구조이다. Next Header 필드와 함께 이해한다.
이동 IP노드가 다른 네트워크로 이동해도 기존 통신 관계를 유지하도록 지원하는 개념이다. 홈 주소, 임시 주소, 에이전트 개념과 연결될 수 있다.
터널링원래 패킷을 다른 패킷의 데이터처럼 감싸 전달하는 방식이다. IPv6 over IPv4 같은 전환 구간이나 가상 경로 구성에서 설명된다.
IP 터널링IP 패킷을 다시 IP 패킷 안에 넣어 전달하는 방식이다. 캡슐화와 디캡슐화를 함께 외운다.
ARP같은 링크에서 IP 주소에 해당하는 MAC 주소를 알아내기 위한 프로토콜이다. ARP 요청은 브로드캐스트 성격으로 설명되는 경우가 많다.
ARP 캐시한 번 확인한 IP-MAC 대응 정보를 일정 시간 저장해 반복 요청을 줄이는 표이다.
RARPMAC 주소를 이용해 IP 주소를 알아내는 방향으로 설명된다. ARP와 방향 비교가 시험 포인트다.
ICMPIP 전달 중 오류 상황을 알리거나 진단 메시지를 전달하는 제어 메시지 프로토콜이다. Echo Request/Reply, Destination Unreachable, Time Exceeded가 대표적이다.
HELLO/ECHO상대 호스트의 존재나 응답 가능성을 확인하는 제어 메시지 성격으로 정리한다. 시험에서는 도달 가능성, 응답 확인, ICMP 키워드를 포함한다.
IGMP호스트와 라우터 사이에서 멀티캐스트 그룹 가입 정보를 관리하는 프로토콜이다. ICMP와 목적이 다르다.
멀티캐스트하나의 송신자가 특정 그룹에 속한 여러 수신자에게 데이터를 보내는 방식이다. IGMP는 이 그룹 관리와 연결된다.

8. 모두 고르기 예상 선지

옳은 선지로 만들기 쉬운 문장

  • IPv6 주소는 128비트이다.
  • IPv6는 확장 헤더를 통해 부가 기능을 연결할 수 있다.
  • ARP는 IP 주소에 대응하는 MAC 주소를 찾는다.
  • ICMP Echo 메시지는 응답 가능성 확인에 활용될 수 있다.
  • IGMP는 멀티캐스트 그룹 관리와 관련된다.

함정 선지로 나오기 쉬운 문장

  • ARP가 MAC 주소로 IP 주소를 찾는다고 하는 선지
  • IGMP를 오류 보고 전용 프로토콜이라고 하는 선지
  • IPv6 주소 길이를 32비트라고 하는 선지
  • 터널링을 ACK를 데이터 프레임에 같이 싣는 피기배킹과 혼동하는 선지

9. 설명형 예상문제와 모범답안

HELLO/ECHO 패킷에 대하여 설명하시오.

상대 장비의 존재, 도달 가능성, 응답 여부를 확인하기 위한 제어 메시지 성격의 패킷이다. ICMP Echo 요청과 응답처럼 네트워크 진단에 활용될 수 있다.

핵심 키워드: 도달 가능성, 응답, 제어 메시지, ICMP

ARP와 RARP를 비교하시오.

ARP는 알고 있는 IP 주소를 이용해 MAC 주소를 찾고, RARP는 알고 있는 MAC 주소를 이용해 IP 주소를 찾는다. 두 프로토콜은 주소 변환 방향이 반대이다.

핵심 키워드: ARP, RARP, IP 주소, MAC 주소

터널링을 설명하시오.

원래 패킷을 다른 프로토콜 패킷 안에 캡슐화해 전달하는 방식이다. 서로 다른 IP 버전 구간을 지나거나 논리적인 통로를 만들 때 사용될 수 있다.

핵심 키워드: 터널링, 캡슐화, IPv4, IPv6

10. 최종 암기 요약

  • IPv6는 128비트, 확장 헤더, Hop Limit를 묶어서 외운다.
  • ARP는 IP→MAC, RARP는 MAC→IP다.
  • ICMP는 오류·진단, IGMP는 멀티캐스트다.
  • 터널링은 캡슐화해서 통과시키는 방식이다.
