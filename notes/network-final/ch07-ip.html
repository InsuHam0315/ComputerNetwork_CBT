1. 챕터 개요

IP 프로토콜 장에서는 네트워크 계층의 큰 역할과 IP 패킷 전달 방식을 함께 본다. 데이터 링크 계층이 한 링크 안의 전달이라면, 네트워크 계층은 여러 네트워크를 지나 목적지 호스트까지 가는 전달을 다룬다.

IP는 비연결형 최선형 전달 성격으로 정리한다. 즉, IP 자체가 모든 패킷의 순서와 신뢰성을 완벽히 보장한다고 외우면 틀리기 쉽다. 신뢰성은 TCP 같은 상위 계층 기능과 함께 설명된다.

시험에서는 라우팅, 혼잡 제어, 거리 벡터, 링크 상태, 외부 라우팅, IP 헤더, 패킷 분할, DHCP가 빈칸·단답·비교형으로 변형되기 쉽다.

2. 시험에 나오는 핵심

  • 네트워크 계층은 호스트 간 패킷 전달과 경로 선택을 담당한다.
  • 비연결형 서비스는 각 패킷을 독립적으로 전달하고, 연결형 서비스는 논리 연결을 설정한 뒤 전달한다.
  • 라우팅은 경로 결정, 포워딩은 실제 다음 홉으로 보내는 동작으로 구분할 수 있다.
  • 거리 벡터는 이웃과 거리 정보를 교환하고, 링크 상태는 토폴로지 정보를 바탕으로 경로를 계산한다.
  • IP 헤더에는 주소, TTL, 프로토콜, 식별자, 플래그, 오프셋 등 전달·분할에 필요한 정보가 들어간다.
  • DHCP는 IP 주소와 네트워크 설정 정보를 자동으로 제공한다.

3. 핵심 개념 상세 설명

네트워크 계층송신 호스트에서 수신 호스트까지 패킷을 전달하는 계층이다. 경로 선택, 논리 주소, 혼잡 제어와 관련된다.
패킷/데이터그램네트워크 계층에서 다루는 전송 단위이다. IP에서는 각 데이터그램이 독립적으로 전달될 수 있다.
연결형 서비스데이터 전달 전에 논리적 연결을 설정하고 그 연결 상태를 기반으로 전달한다. 순서 관리와 상태 관리가 강조된다.
비연결형 서비스연결 설정 없이 각 패킷을 독립적으로 전달한다. IP의 기본 성격을 설명할 때 자주 쓰인다.
라우팅목적지까지 어떤 경로를 사용할지 결정하는 기능이다. 라우팅 테이블, 비용, 라우팅 프로토콜과 함께 이해한다.
포워딩라우팅 결과를 바탕으로 들어온 패킷을 어느 인터페이스 또는 다음 홉으로 내보낼지 처리하는 동작이다.
혼잡 제어네트워크 내부에 패킷이 과도하게 몰려 지연과 손실이 커지는 상황을 완화하는 기능이다. 흐름 제어와 구분해야 한다.
거리 벡터각 라우터가 이웃 라우터와 목적지까지의 거리 정보를 교환하며 경로를 갱신하는 방식이다.
링크 상태각 라우터가 링크 상태 정보를 수집·전파하고 전체 토폴로지 기반으로 최단 경로를 계산하는 방식이다.
외부 라우팅서로 다른 자율 시스템 사이의 경로 정보를 교환하는 라우팅이다. 내부 라우팅과 범위가 다르다.
IP 헤더출발지/목적지 IP 주소, TTL, 프로토콜, 헤더 길이, 전체 길이, 식별자, 플래그, 단편 오프셋 등으로 구성된다.
TTL패킷이 네트워크 안에서 무한히 순환하지 않도록 홉을 지날 때마다 감소하는 값이다.
패킷 분할IP 패킷이 다음 링크의 MTU보다 클 때 여러 조각으로 나누는 과정이다. 식별자, 플래그, 오프셋이 함께 출제된다.
MTU한 링크에서 한 번에 실을 수 있는 최대 전송 단위이다. MTU보다 큰 IP 패킷은 분할이 필요할 수 있다.
DHCP호스트가 IP 주소, 서브넷 마스크, 기본 게이트웨이, DNS 서버 같은 설정을 자동으로 받게 하는 프로토콜이다.

5. 계층별 기능 정리

데이터 링크 계층과의 차이데이터 링크 계층은 인접 노드 사이 전달이고, 네트워크 계층은 여러 링크를 거친 종단 간 전달이다.
전송 계층과의 차이네트워크 계층은 호스트까지 보내고, 전송 계층은 포트 번호를 이용해 프로세스까지 전달한다.
라우터의 역할라우터는 네트워크 계층 주소와 라우팅 테이블을 이용해 패킷의 다음 경로를 결정한다.

8. 모두 고르기 예상 선지

옳은 선지로 만들기 쉬운 문장

  • 네트워크 계층은 목적지까지의 경로 선택과 패킷 전달을 담당한다.
  • IP는 기본적으로 비연결형 최선형 전달로 설명된다.
  • 거리 벡터 방식은 이웃과 거리 정보를 교환한다.
  • 링크 상태 방식은 토폴로지 정보를 바탕으로 경로를 계산한다.
  • DHCP는 호스트의 IP 설정 자동화와 관련된다.

함정 선지로 나오기 쉬운 문장

  • IP가 모든 패킷의 신뢰성과 순서를 항상 보장한다고 하는 선지
  • 라우팅과 포워딩을 완전히 같은 동작으로만 설명하는 선지
  • 혼잡 제어를 수신자 버퍼 하나만 보호하는 기능이라고 하는 선지
  • DHCP를 MAC 주소 변환 전용 프로토콜이라고 하는 선지

9. 설명형 예상문제와 모범답안

거리 벡터와 링크 상태 라우팅을 비교하시오.

거리 벡터는 이웃 라우터와 목적지까지의 거리 정보를 교환해 경로를 갱신한다. 링크 상태는 각 링크의 상태 정보를 모아 토폴로지를 구성하고 최단 경로를 계산한다.

핵심 키워드: 거리 벡터, 링크 상태, 이웃, 토폴로지

IP 패킷 분할을 설명하시오.

패킷 크기가 다음 링크의 MTU보다 크면 여러 조각으로 나누어 전달한다. 각 조각은 식별자, 플래그, 단편 오프셋 등의 정보를 이용해 원래 패킷으로 재조립될 수 있다.

핵심 키워드: MTU, 분할, 식별자, 오프셋

DHCP의 기능을 설명하시오.

DHCP는 호스트가 네트워크에 접속할 때 IP 주소와 서브넷 마스크, 기본 게이트웨이, DNS 서버 같은 설정 정보를 자동으로 받게 한다.

핵심 키워드: DHCP, IP 주소, 게이트웨이, DNS

10. 최종 암기 요약

  • IP는 비연결형 최선형 전달이다.
  • 라우팅은 길 찾기, 포워딩은 다음 홉으로 보내기다.
  • 거리 벡터는 이웃과 거리, 링크 상태는 토폴로지다.
  • 분할은 MTU, 식별자, 플래그, 오프셋을 묶어서 외운다.
