1. 챕터 개요

MAC 계층은 같은 전송 매체를 여러 장치가 공유할 때 누가 언제 프레임을 보낼지 정하는 부분이다. 시험에서는 MAC을 단순히 주소 이름으로만 보지 말고, 매체 접근 제어와 프레임 전달 규칙을 함께 묶어서 이해해야 한다.

데이터 링크 계층은 설명할 때 LLC와 MAC으로 나누어 보는 경우가 많다. LLC는 상위 계층에 논리적인 링크 서비스를 제공하고, MAC은 실제 LAN 매체에서 프레임을 보내기 위한 주소, 접근 방식, 충돌 처리와 관련된다.

이 장은 빈칸형으로 IEEE 802, MAC, LLC, CSMA/CD, 충돌 감지, 토큰 버스, 토큰 링, FCS 같은 용어가 자주 나올 수 있다. 설명형으로는 허브와 스위치 허브의 차이, CSMA/CD 과정, 토큰 방식의 특징을 정리해 두면 좋다.

2. 시험에 나오는 핵심

  • IEEE 802는 LAN/MAN 표준을 다루는 계열이며, 이더넷 계열은 그중 가장 대표적인 LAN 기술로 이해한다.
  • MAC은 물리 주소 기반의 프레임 전달과 매체 접근 제어를 담당한다.
  • LLC는 매체 종류와 독립적으로 상위 계층에 논리 링크 서비스를 제공하는 역할로 설명된다.
  • CSMA/CD는 송신 전 매체 감지, 송신 중 충돌 감지, 충돌 후 백오프와 재전송 순서로 정리한다.
  • 토큰 버스와 토큰 링은 토큰을 가진 노드만 송신할 수 있는 권한 기반 접근 방식이다.
  • 프레임 오류는 FCS/CRC 같은 오류 검출 정보로 확인하고, 상위 절차 또는 링크 제어 절차에서 복구한다.

3. 핵심 개념 상세 설명

IEEE 802근거리 통신망과 도시권 통신망을 표준화하는 IEEE 표준 계열이다. 시험에서는 IEEE 802를 LAN 표준과 연결하고, MAC/LLC 구분과 함께 묻는 경우가 많다.
LAN비교적 제한된 공간에서 고속으로 장치들을 연결하는 네트워크이다. LAN에서는 이더넷, 토큰 링 같은 MAC 기술과 스위치 장비가 자주 함께 등장한다.
MACMedium Access Control의 약자로, 프레임을 언제 전송할지와 어떤 물리 주소를 기준으로 전달할지 다룬다. IP 주소와 달리 같은 링크 안에서 실제 프레임 전달에 직접 관여한다.
LLCLogical Link Control의 약자로, MAC 위에서 상위 계층에 공통적인 논리 링크 서비스를 제공한다. 시험에서는 MAC은 매체 접근, LLC는 논리 링크라는 식으로 대비된다.
MAC 주소네트워크 인터페이스를 식별하는 물리 주소이다. 스위치는 MAC 주소를 학습해 포트별 전달을 수행하고, ARP는 IP 주소에 대응하는 MAC 주소를 찾는다.
이더넷 프레임목적지 MAC, 출발지 MAC, 유형 또는 길이, 데이터, FCS 등을 포함한다. 프레임 구조 문제에서는 각 필드가 어느 기능과 연결되는지 묻기 쉽다.
CSMA/CDCarrier Sense Multiple Access with Collision Detection이다. 공유 매체에서 먼저 듣고, 비어 있으면 보내고, 충돌이 나면 감지해 멈춘 뒤 일정 시간 후 다시 보내는 방식이다.
충돌 감지두 노드 이상이 동시에 전송해 신호가 겹치는 상황을 확인하는 기능이다. 충돌이 감지되면 전송을 계속 밀어붙이는 것이 아니라 중단과 백오프가 이어진다.
백오프충돌 후 모든 노드가 동시에 다시 보내지 않도록 재전송 대기 시간을 분산하는 과정이다. 충돌이 반복될수록 대기 범위가 커지는 방식과 연결된다.
토큰 버스물리적으로는 버스 형태처럼 보여도 논리적으로 토큰 전달 순서를 만들어 송신 권한을 제어하는 방식이다.
토큰 링링 구조에서 토큰이 순환하고, 토큰을 가진 노드가 데이터를 전송한다. 토큰 관리, 토큰 분실, 순서 제어가 시험 포인트다.
허브들어온 신호를 여러 포트로 단순 중계하는 장비로, 충돌 영역이 공유될 수 있다. 스위치와 비교형으로 자주 나온다.
스위치MAC 주소 테이블을 학습하고 목적지에 해당하는 포트로 프레임을 선택 전달한다. 허브보다 불필요한 전송과 충돌 가능성을 줄인다.
FCS/CRC프레임 끝에 붙는 오류 검출 정보이다. 수신자는 계산 결과와 수신된 검사 값을 비교해 프레임 오류를 판단한다.

5. 계층별 기능 정리

데이터 링크 계층 관점MAC은 데이터 링크 계층의 하위 기능으로 프레임 단위의 지역 전달과 매체 접근을 담당한다.
물리 계층과의 관계물리 계층은 비트 신호를 보내지만, MAC은 그 비트들을 프레임으로 보고 충돌과 접근 순서를 제어한다.
네트워크 계층과의 관계네트워크 계층은 IP 주소와 패킷 전달을 다루고, MAC 계층은 같은 링크 안에서 MAC 주소를 기준으로 프레임을 전달한다.

8. 모두 고르기 예상 선지

옳은 선지로 만들기 쉬운 문장

  • MAC은 매체 접근 제어와 물리 주소 기반 프레임 전달에 관여한다.
  • LLC는 상위 계층에 논리 링크 서비스를 제공하는 기능으로 설명된다.
  • CSMA/CD는 충돌 발생 시 백오프 후 재전송을 시도할 수 있다.
  • 스위치는 MAC 주소 테이블을 이용해 필요한 포트로 프레임을 전달할 수 있다.
  • 토큰 방식에서는 토큰 보유 여부가 송신 권한과 연결된다.

함정 선지로 나오기 쉬운 문장

  • MAC 주소를 IP 주소와 같은 네트워크 계층 주소라고 단정하는 선지
  • 허브가 MAC 주소를 학습해 지능적으로 포워딩한다고 하는 선지
  • CSMA/CD에서 충돌 후 즉시 모든 노드가 동시에 재전송한다고 하는 선지
  • 토큰 방식도 경쟁 기반 충돌 감지만으로 동작한다고 하는 선지

9. 설명형 예상문제와 모범답안

CSMA/CD의 동작 과정을 설명하시오.

송신 전 매체를 감지하고, 매체가 비어 있으면 전송한다. 전송 중 충돌을 감지하면 전송을 중단하고 충돌 사실을 알린 뒤, 백오프 시간을 기다렸다가 재전송한다.

핵심 키워드: 매체 감지, 충돌 감지, 백오프, 재전송

허브와 스위치 허브의 차이를 설명하시오.

허브는 들어온 신호를 단순히 여러 포트로 반복 전달하므로 충돌 영역을 공유하기 쉽다. 스위치는 MAC 주소를 학습해 목적지 포트로 선택 전달하므로 불필요한 전송을 줄인다.

핵심 키워드: 허브, 스위치, MAC 주소, 선택 전달

MAC과 LLC의 차이를 설명하시오.

MAC은 매체 접근 제어와 물리 주소 기반 프레임 전달을 담당하고, LLC는 상위 계층에 공통적인 논리 링크 서비스를 제공한다.

핵심 키워드: MAC, LLC, 매체 접근, 논리 링크

10. 최종 암기 요약

  • MAC은 주소 이름만 외우지 말고, 매체 접근 제어까지 함께 외운다.
  • LLC는 논리 링크, MAC은 실제 LAN 매체 접근으로 구분한다.
  • CSMA/CD는 감지 → 전송 → 충돌 감지 → 백오프 → 재전송 순서다.
  • 허브는 단순 중계, 스위치는 MAC 학습 후 선택 전달이다.
