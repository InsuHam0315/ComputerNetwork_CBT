1. 챕터 개요

인접 노드 사이의 프레임 전송에서 흐름 제어, 오류 제어, 재전송 방식을 정리한다.

2. 시험에 나오는 핵심

  • 프레임의 의미와 경계
  • 흐름 제어와 오류 제어 구분
  • 정지-대기와 슬라이딩 윈도우
  • Go-Back-N과 선택적 재전송 비교
  • HDLC/LAP 계열의 링크 제어

3. 핵심 개념 설명

  • 데이터 링크 계층에서 사용하는 전송 단위
  • 송신 속도가 수신 처리 능력을 넘지 않도록 조절하는 기능
  • 손상·분실 프레임을 검출하고 복구하는 기능
  • 여러 프레임을 연속 전송하고 ACK에 따라 범위를 움직이는 방식
  • 한 프레임 전송 후 확인 응답을 기다리는 단순 방식
  • 오류 지점 이후의 프레임을 다시 보낼 수 있는 재전송 방식
  • 오류가 난 프레임만 골라 재전송하는 방식
  • 비트 중심의 대표적인 데이터 링크 제어 프로토콜

4. 주요 프로토콜/용어 정리

프레임데이터 링크 계층에서 사용하는 전송 단위
흐름 제어송신 속도가 수신 처리 능력을 넘지 않도록 조절하는 기능
오류 제어손상·분실 프레임을 검출하고 복구하는 기능
슬라이딩 윈도우여러 프레임을 연속 전송하고 ACK에 따라 범위를 움직이는 방식
정지-대기한 프레임 전송 후 확인 응답을 기다리는 단순 방식
Go-Back-N오류 지점 이후의 프레임을 다시 보낼 수 있는 재전송 방식
선택적 재전송오류가 난 프레임만 골라 재전송하는 방식
HDLC비트 중심의 대표적인 데이터 링크 제어 프로토콜

5. 계층별 기능 정리

  • 인접 노드 사이의 프레임 전송에서 흐름 제어, 오류 제어, 재전송 방식을 정리한다.
  • 하위 계층의 전달 기능을 상위 계층 서비스와 연결해서 이해합니다.

6. 헷갈리는 비교표

정지-대기단순하지만 효율 낮음슬라이딩 윈도우연속 전송으로 효율 높음
Go-Back-N오류 이후 구간 재전송선택적 재전송오류 프레임만 재전송
흐름 제어수신 버퍼 보호오류 제어손상·분실 복구

8. 모두 고르기 예상 선지

  • 흐름 제어에 대한 설명으로 옳은 것을 모두 고르시오.
  • 오류 제어에 대한 설명으로 옳은 것을 모두 고르시오.
  • 슬라이딩 윈도우 방식의 특징으로 옳은 것을 모두 고르시오.
  • 데이터 링크 계층 프로토콜에 대한 설명으로 옳은 것을 모두 고르시오.
